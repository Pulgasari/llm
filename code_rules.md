1. Language & Comments:
   - All code documentation, naming, and comments must be in English.
   - Comments must be entirely lowercase, except for technical acronyms or capitalization for emphasis (e.g., POO, RGBA, FIFO, LIFO).

2. Indentation & Spacing:
   - Use strict tabsize 2 (two spaces for indentation).
   - In single-line procedures, enforce a space between the 'proc' keyword and the parameter block, e.g., 'proc (...)'.

3. Vertical & Horizontal Alignment:
   - Multi-line block structures (structs, unions, etc.) must have vertically aligned colons (':').
   - Single-line assignments or horizontal pairs must use compact spacing, formatted as 'key: value, key: value'.

4. Control Flow Blocks:
   - Short control flow blocks and standard guard clauses/error checks must be kept inline on a single line: 'if (...) { ... }'.

5. Ordering & Structure:
  - Code elements within logical groups where the absolute order does not matter must be sorted strictly alphabetically.
  - Section headers must use short, clean half-length divider lines (e.g., '// === ... ===').
